# README Compare — Delta Tracking Skill

## Purpose

The most compelling README Roast output: a before/after comparison showing exactly how much a README improved. This is the content that goes viral on X/Twitter — "I rewrote my repo's README using patterns from 50 top-starred devtools projects. Here's what changed."

---

## Commands

```
/readme-compare                           # Compare last two audits from history
/readme-compare #1 #3                     # Compare specific audit numbers
/readme-compare <before-file> <after-file> # Compare specific files (legacy)
```

## Workflow

### Step 1: Find Audit Data

**Primary method — read from `.readme-roast/history.json`:**

1. Check if `.readme-roast/history.json` exists in the current repo
2. If yes and has 2+ audits:
   - No arguments: use the last two entries
   - `#N #M` arguments: use audit number N and M from the audits array
   - Load full snapshot data from `.readme-roast/snapshots/{id}.json`
3. If yes but only 1 audit:
   - Tell user: "Only one audit on record. Make README changes, then run `/readme-audit` again."
4. If `.readme-roast/` doesn't exist:
   - Fall back to legacy method below

**Legacy fallback — read from report files:**

1. Look in the current directory for `README-AUDIT-REPORT*.md` files
2. Sort by date (from filename or file modified date)
3. Use oldest as "before", newest as "after"
4. If only one file exists, tell user to run another audit after making changes

If specific files are provided as arguments:
1. Read both files
2. Parse scores from each

### Step 2: Parse Both Audits

Extract from each audit (from snapshot JSON or parsed markdown):
- Overall README Score
- Per-category scores (6 categories)
- Pattern flags (has_gif, has_comparison, etc.)
- Star killers list
- Action items
- Star count (if available)
- Changes made between audits (from `changes_since_last`)

### Step 3: Calculate Deltas

For each metric:
- Delta = After - Before
- Trend: ▲ (improved), ▼ (declined), ── (unchanged)
- Percentage change

### Step 4: Generate Comparison Report

Output to `README-COMPARE.md`

## Delta Interpretation

| Delta | Symbol | Meaning |
|-------|--------|---------|
| +10 or more | ▲▲ | Major improvement |
| +1 to +9 | ▲ | Improvement |
| 0 | ── | No change |
| -1 to -9 | ▼ | Slight decline |
| -10 or more | ▼▼ | Significant decline — investigate |

## Output

Save to `README-COMPARE.md` in the current working directory.
